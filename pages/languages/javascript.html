Know your variables

Variables are used to store data values. In JavaScript, you can declare variables using var, let, or const. The most common way to declare a variable is by using let and const.

  • let: Used to declare a variable that can be reassigned later. 
    let age = 25; // You can change the value of age later
  • const: Used to declare a variable that cannot be reassigned. It must be initialized with a value. 
    const pi = 3.14; // You cannot change the value of pi later

Case Sensitivity in Variables

In JavaScript all variables and function names are case sensitive. MYVAR is not the same as MyVar nor myvar.
Example:
var someVariable;
var anotherVariableName;
var thisVariableNameIsSoLong;

Uninitialized Variables

When JavaScript variables are declared, they have an initial value of undefined. If you do a mathematical operation on an undefined variable your result will be NaN which means "Not a Number". If you concatenate a string with an undefined variable, you will get a string of undefined.

Data Types in JavaScript

JavaScript has several built-in data types that you can use to represent different kinds of values. The main data types in JavaScript are:

  • Number: Represents numeric values, both integers and floating-point numbers. 
    let age = 25; // A number variable
  • String: Represents text values enclosed in single or double quotes. 
    let name = "John"; // A string variable
  • Boolean: Represents logical values, either true or false. 
    let isStudent = true; // A boolean variable
  • Array: Represents a list of values enclosed in square brackets. 
    let hobbies = ["reading", "coding", "gaming"]; // An array variable
  • Object: Represents a collection of key-value pairs enclosed in curly braces. 
    let person = { firstName: "Jane", lastName: "Doe" }; // An object variable
  • null: Represents the intentional absence of any object value. 
    let emptyValue = null; // A null variable
  • undefined: Represents a variable that has been declared but not assigned a value. 
    let uninitializedValue; // An undefined variable

You can use these data types to store and manipulate different kinds of information in your JavaScript programs. Understanding data types is essential for effective programming and helps you choose the right type for your variables based on the kind of data you want to work with.

DOM Manipulation

The Document Object Model (DOM) allows JavaScript to interact with HTML elements on a web page.

Selecting Elements

You can select HTML elements using various methods:

Fetch API Example

The Fetch API provides a modern way to make HTTP requests:

// GET request
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

// POST request
fetch('https://api.example.com/data', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({ key: 'value' })
})
  .then(response => response.json())
  .then(data => console.log(data));

ES6+ Modules

ES6 modules allow you to organize code into reusable pieces and manage dependencies effectively.

Import and Export

Use export to expose functions, variables, or classes from a module, and import to use them in other files.

// math.js - Named exports
export function add(a, b) {
  return a + b;
}

export function multiply(a, b) {
  return a * b;
}

export const PI = 3.14159;

// Default export
export default function subtract(a, b) {
  return a - b;
}

// main.js - Importing
import subtract, { add, multiply, PI } from './math.js';
import * as mathUtils from './math.js';

console.log(add(2, 3)); // 5
console.log(multiply(4, 5)); // 20
console.log(PI); // 3.14159
console.log(subtract(10, 3)); // 7

// Using namespace import
console.log(mathUtils.add(1, 2)); // 3
Module Benefits

Modules help you:

  • Organize code: Split functionality into logical units
  • Encapsulation: Keep implementation details private
  • Reusability: Share code across multiple files
  • Dependency management: Clearly define what each module needs
  • Tree shaking: Bundlers can remove unused code
  • Namespace pollution: Avoid global variable conflicts
Dynamic Imports

ES2020 introduced dynamic imports for loading modules conditionally:

// Dynamic import
async function loadMath() {
  const mathModule = await import('./math.js');
  return mathModule.add(2, 3);
}

// Conditional loading
if (someCondition) {
  import('./heavy-library.js')
    .then(module => {
      module.doSomething();
    });
}
